Зарегестрированные пользователи ( держатели Бон-карт) в личном кабинете могут получать уведомления ( текстовую информацию) от системы ( или можно сказать от администратора ). Т.к. личный кабинет - отдельная система ( со своей базой данных ), то необходимо сделать в основной базе данных следующий функционал:
    1.  получить возможность по номеру карты получать новые ( не переданные в базу личного кабинета) сообщения, адресованные данной бон-карте
    2. устанавливать для новых сообщений статус "не новое" ( то есть прочитанное системой личного кабинета, и сохраненное в своей базе данных)

Личные данные пользователя:
- Данные пользователя
	>>> Бон-карта действительна до: (дата, до которой действительна карта)
	Клуб: ( накопилка, персона - название клуба )
		bc_admin.VC_CLUB_CARD_CLUB_ALL.NAME_CARD_STATUS
	Категория Бон-карты ( номер категории)
		bc_admin.VC_CLUB_CARD_CLUB_ALL.NAME_BON_CATEGORY
		bc_admin.VC_CLUB_CARD_CLUB_ALL.NAME_DISC_CATEGORY
	
	>>> Пересчет категории через: (кол-во дней до пересчета)
		

- Бонусные средства:
	Доступно бонов на карте: 
		bc_admin.VC_CLUB_CARD_CLUB_ALL.BAL_CUR_FMRT

	Накоплено бонов на карте:
		bc_admin.VC_CLUB_CARD_CLUB_ALL.BAL_ACC_FMRT

	>>>Потрачено бонов:
		bc_admin.VC_CLUB_CARD_CLUB_ALL.

- Дополнительная информация:
	Время пользования бон-картой: ( наверное, дата со дня выдачи/первой транзакции карты)
		(bc_admin.VC_CLUB_CARD_CLUB_ALL.DATE_OPEN) : дата открытия

- Информация о покупках:
	>>>Последняя покупка: ( адрес места, в котором была совершена покупка)
	>>>На сумму: ( сумма последней покупки )
	>>>Потрачено бонов: ( кол-во начисленных бонов за покупку)
	>>>Начислено бонов: ( кол-во потраченных бонов за покупку )

- История покупок:
	>>> Чаще всего вы покупаете: ( товар, который приобретается держателем карты чаще всего)
	>>> Больше всего покупок Вы совершаете: (адрес, по которому совершается самое большое кол-во покупок)
